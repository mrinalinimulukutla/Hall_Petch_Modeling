# Agent instructions for `Physics_informed_ML_Hall_Petch`

Read this first. It tells an AI agent (Claude Code, Cursor, etc.) the project
state, conventions, and gotchas so it can produce useful work without breaking
things. Human contributors should also read it — it complements `README.md`
(navigation) and `CONTRIBUTING.md` (workflow).

## 1. What this project is

Companion repository for an *Acta Materialia* manuscript on a best-practices
framework for ML prediction of yield strength (YS) and Vickers hardness (HV)
in FCC HEAs (94 alloys, Al–Co–Cr–Cu–Fe–Mn–Ni–V). ML is used both to construct
models and to audit physics-informed ones; the analysis climbs a five-tier
"staircase" of model flexibility validated cluster-out at every step.

Three audiences for the artifacts:

- **Paper** (`paper/main.tex` + `paper/supplementary.tex`) — primary
  publication; canonical narrative
- **Notebook** (`notebook/Hall_Petch_HEA_Analysis.ipynb`) — interactive
  reproduction
- **Word report** (`report/Comprehensive_Analysis_Report.docx`) — long-form
  internal document

All three are aligned on the same numbers. **If you edit one, audit the other
two for the same change.** Failure to do so is the #1 source of regressions.

## 3. Canonical state — headline numbers

If your work changes any of these, also update the corresponding paragraph in
the paper AND `notebook/_generate_notebook.py` AND `report/generate_report.py`.

**Dataset**

- 94 alloys total: 93 with YS, 94 with HV
- B-campaign (BBA+BBB+BBC) = 34 alloys, fixed processing (CW 60 %, 950 °C);
  C-campaign (CBA+CBB+CBC) = 59 alloys, swept 675–1250 °C
- YS 152–544 MPa, HV 58–228, grain size 15–212 μm

**Tier 1 — grain size**

- YS classical HP LOO R² = 0.406; top-4 laws ΔBIC < 2; n_opt = 0.548;
  Bayesian posterior n = 0.53 ± 0.16
- HV classical HP: 5-fold 0.09 / LOO 0.136 / LOBO −0.08

**Tier 3 — composition HP (M-models)**

- M3: LOO 0.652 / LOBO 0.625 / ext 133 MPa / stacking weight 0.902;
  k_HP = 766 MPa·μm^(1/2); α_V = +291 MPa; M1 vs M3 ΔBIC = 1.4
- M15 (= M3 + SD_grain + SD_grain·d⁻¹ᐟ²): LOO = LOBO = 0.694, stacking
  weight 0.89, Δelpd = 11.7 over M3
- Per-alloy k_eff vs composition: R² = 0.006 (composition-independent slope
  in this dataset)
- HV: every composition M-model LOO ≤ 0.14; M0 baseline is best

**Tier 4 — non-linear ML**

- Legacy panel: XGBoost LOO 0.729 → LOBO 0.574 (the validation failure mode)
- ARMOTE-CV best YS: Bayesian ridge @ S2, LOBO 0.613; HV: nothing >0.06 LOBO
- Fair comparison: S1→S2 (+Wen) lifts LOBO ~0.39→~0.6 for linear AND
  non-linear; best linear LOBO 0.62 ≥ best non-linear 0.615

**Tier 5 — symbolic regression**

- PySR YS F3+O3 accuracy: LOO 0.765 / LOBO 0.668 / BIC 690 / ext 120 MPa, safe
- PySR YS elbow (2 const): σy = (SD_grain + 1016.5)(d⁻¹ᐟ² + 0.306/Ω),
  LOO 0.674 / LOBO 0.555
- SISSO Full: LOO 0.665, RMSE 46.9 MPa, LOBO 0.380, BIC 714, ext 421 MPa
  (σ_χ²/δ_μ singularity — do NOT deploy)
- SISSO Robust: LOO 0.609, RMSE 50.7, LOBO 0.358, BIC 717, ext 163 MPa, safe,
  no imputation
- SISSO + SD_grain (YS): LOO 0.560 / LOBO 0.429 / ext 128, safe
- HV elbow (3 const): HV ≈ 221.5 − 83.9(6.93−d)/SD_grain + ΔH_mix/t_hold²,
  refit LOO 0.727 / LOBO 0.636 — the only cross-cluster HV survivor
- No HV model generalizes externally (division singularities, ext 10³–10⁴)

**Hardness–strength (§4.7)**

- Tabor C_eff = 5.13 ± 1.36; n_eff ≈ 0.15
- HV–YS ranks: global ρ = 0.46; within-batch 0.70–0.95; B-pooled 0.95;
  C-pooled 0.09; partial ρ(HV,YS|d) = 0.24 — composition (V) is the primary
  scrambler; grain size is a confounder
- log(C_eff) = 1.36 + 0.10·log d − 2.06·x_V

If a script run reports one of these numbers differing by more than 1 %,
**stop and investigate before editing the paper**.

## 4. Path conventions — never hardcode

Every script imports paths from `scripts/_config.py`. **Never write
`'/Users/...'`** in any committed file.

```python
# Script inside a tier folder (one level below scripts/)
import sys, os
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
from _config import REPO_ROOT, DATA_DIR, RAW_DATA_DIR, RESULTS_DIR, PLOTS_DIR, PAPER_DIR, PAPER_FIG_DIR
```

`_config.py` auto-appends every tier folder to `sys.path`, so cross-script
imports (`from external_validation import ...`) work from any folder as long
as `_config` is imported first. Use `find_script('name')` to locate a script
programmatically (the notebook generator does this).

## 5. Files that must stay in sync

**Updating a model result (e.g., SISSO numbers)**
1. Re-run the script (see the folder READMEs or `make help` for the mapping)
2. Update the paper: search `paper/main.tex` for the old number; replace.
3. Update `report/generate_report.py` and `notebook/_generate_notebook.py`;
   re-run both generators.
4. Re-build the paper PDF; delete LaTeX aux files before committing.
5. If the change touches M3 or the HV baseline, re-run
   `scripts/03_tier3_composition_hp/export_fitted_models.py` so the pickles
   match — CI asserts canonical values from the pickles.

**Adding a new analysis script**
- [ ] Lives in the tier folder matching its role; two-level sys.path bootstrap
- [ ] Imports paths from `_config.py`
- [ ] Writes CSV to `RESULTS_DIR`, plots to `PLOTS_DIR` with the next NN_ number
- [ ] Reports BOTH LOO and LOBO wherever it reports any cross-validated metric
- [ ] Symbolic equations: run the singularity audit before any deployment claim
- [ ] Adds a row via `06_validation_floor/unified_model_table.py`, not a
      parallel table
- [ ] Adds a regression test under `tests/` locking the canonical output

## 6. Writing style for the paper

Run `slop-detect paper/main.tex` (user's PATH) before claiming a prose edit is
done. No bold/italic paragraph lead-ins; avoid `Importantly,`/`Crucially,`/
`leverage`/`exhibit`/`utilize`; tie every claim to a number; `Notably,` and
concessive `Although X, Y` constructions are house style. Validate new
references with `bib-check-cli` against CrossRef before adding to
`references.bib`.

## 7a. Metallurgical constants — do not "fix"

These look wrong but are correct. Do not change them without re-running the
regression tests.

- **Co shear modulus = 75 GPa** (not 82). Verified against FCC-Co literature.
- **Fe a_FCC (γ-Fe) = 3.590 Å** (not 3.515 = δ-Fe).
- **Mn radius = 127 pm** (not Goldschmidt 137 pm) — effective FCC-SS value
  per Toda-Caraballo.
- **`Phi_VLC` uses sphere volumes** as an intentional correlation proxy; the
  VLC model itself uses FCC lattice volumes. Do NOT "fix" Phi_VLC — it breaks
  XGBoost.

## 8. Gotchas

- **Do not edit `Hall_Petch_HEA_Analysis.ipynb` directly** — regenerate via
  `notebook/_generate_notebook.py`.
- **Do not commit LaTeX aux files** (`*.aux`, `*.bbl`, `*.blg`, `*.log`,
  `*.out`, `*.spl`, `*.toc`); run the cleanup after every compile.
- **Do not commit the .docx or PDFs unless the source changed** — binary
  noise in diffs.
- **ARMOTE raw outputs live in a separate archive** (~1.5 GB); this repo
  keeps only `results/armote_ladder_cv.csv`. Do not re-add the raw folders.
- **Cited-literature PDFs are deliberately absent** (copyright); do not add
  them. `paper/references.bib` is the source of record.
- **The full reproduction (~85 min)** is documented in `docs/reproducing.md`;
  follow the stage order rather than running scripts ad hoc.

## 9. When in doubt

- Read the relevant section of `paper/main.tex`
- Check `docs/reproducing.md` for the canonical run order
- Read `git log --oneline` to see what recently changed and why
- Open an issue with the `question` label rather than guessing
